# Contributing to DAPPR

Thank you for considering contributing to DAPPR! We appreciate your time and effort in helping us improve this project.

## Code of Conduct

This project and everyone participating in it is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How to Contribute

### Reporting Bugs

1. **Check for Existing Issues**: Before creating a new issue, please check if a similar issue already exists.
2. **Create a New Issue**: If you find a bug, please create a new issue with a clear title and description.
3. **Provide Details**: Include steps to reproduce the issue, expected behavior, and actual behavior.

### Suggesting Enhancements

1. **Check Existing Suggestions**: Look for similar enhancement requests before creating a new one.
2. **Create a Feature Request**: Clearly describe the enhancement you'd like to see.
3. **Explain the Benefits**: Explain why this enhancement would be valuable to the project.

### Making Changes

1. **Fork the Repository**: Create a fork of the repository.
2. **Create a Branch**: Make your changes in a new git branch.
3. **Commit Your Changes**: Write clear, concise commit messages.
4. **Push to Your Fork**: Push your changes to your fork.
5. **Submit a Pull Request**: Open a pull request with a clear description of your changes.

### Code Style

- Follow the existing code style.
- Run `cargo fmt` before committing Rust code.
- Use meaningful commit messages.
- Keep PRs focused and limited to a single feature or bug fix.

## Development Setup

1. Fork and clone the repository
2. Install dependencies (see README.md)
3. Create a branch for your changes
4. Make your changes
5. Run tests
6. Submit a pull request
